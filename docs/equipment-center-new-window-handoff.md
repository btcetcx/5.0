# 设备中心新窗口交接说明

日期：2026-05-31

## 任务目标

你负责把「设备中心」完整迁移并完善到 `vue3` 标准工程里，交付一个可以正常运行、可以预览、可以继续对接后端契约的设备中心。

设备中心当前没有独立的原始 `equipment-list.jsx`。因此本任务的权威入口来自当前 Vue 顶部导航中已经预设的设备中心分类。页面能力需要参考主流设备管理系统（EAM/CMMS）的常见业务逻辑来补齐，但不能改变当前预设分类，不能擅自扩展新的二级模块。

遇到以下情况必须先问用户：

- 想新增当前导航没有的二级分类。
- 想删除或重命名当前已经预设的分类。
- 想把设备中心和财务、能耗、采购、仓储做深度联动。
- 不确定某个字段是否属于设备管理、生产管理、仓储管理或能耗管理。
- 需要决定审批流、资产折旧、IoT 实时采集、维保计费等超出当前页面范围的规则。

## 参考标准

参考主流设备管理系统的能力时，只吸收以下稳定业务模式：

- 设备资产全生命周期：建档、启用、运行、停机、维修、保养、报废。
- 预防性维护：按周期、运行小时、点检结果或手动计划生成保养任务。
- 维修工单：报修、派工、处理、验收、关闭。
- 点检巡检：点检计划、点检项目、执行记录、异常升级维修。
- 备件管理：备件库存、领用申请、维修消耗、低库存预警、采购申请。
- 设备状态看板：运行、停机、故障、保养中、维修中、闲置、报废。

不要加入以下复杂能力，除非用户另行确认：

- IoT 实时曲线和设备联网采集。
- 折旧、资产会计、财务凭证。
- 能耗分析。
- OEE 深度计算。
- 维保合同、外包服务商结算。
- 手机扫码点检。

## 联动边界

第一阶段允许做的联动：

- 点检异常生成维修单。
- 维修使用备件减少备件可用库存。
- 保养使用备件生成领用记录。
- 保养完成回写设备最近保养。
- 点检完成回写设备最近点检。
- 维修验收通过回写设备状态。

第一阶段只展示、不深接的联动：

- 生产工单使用设备。
- 质检异常要求设备点检。
- 仓储备件库存真实扣减。
- 采购补货单真实创建。
- 财务维修费用或资产折旧。
- 能耗实时监测。

如果需要从“展示/前端状态”升级成真实跨模块接口，必须先问用户。
