# 新窗口通用守则

日期：2026-05-31

这份守则用于所有新会话窗口。当前项目已进入「验收调整阶段」，任何模块开工前都必须先读，不允许跳过。

## 一句话原则

先看母版，再看现有 Vue，再看验收意见，再动手。  
不确定就问，不允许自己发挥。

## 开工前必须读取

每个新窗口开工前，必须先读以下内容：

1. 母版库
   - `vue3/src/views/template-gallery/TemplateGallery.vue`

2. 公共组件目录
   - `vue3/src/components/list-page`
   - `vue3/src/components/detail-page`
   - `vue3/src/components/form-page`
   - `vue3/src/components/setting-page`

3. 当前模块导航
   - `vue3/src/layouts/erp-shell/navigation.ts`

4. 当前模块路由
   - `vue3/src/app/router/index.ts`

5. 当前模块已确认交接文档、验收记录和用户最新意见
   - 当前阶段不再要求参考原 JSX 页面。
   - 以当前 Vue 已完成页面、母版库、用户最新验收意见为准。
   - 如果历史交接文档和用户最新意见冲突，以用户最新意见为准。

6. 当前模块已有 Vue 页面、mock、types、resources
   - `vue3/src/views/<module>`
   - `vue3/src/app/api/<module>`
   - `vue3/src/mock/<module>`

## 母版优先级

页面形态必须优先复用母版：

- 列表页：`AwListPage`、`AwListToolbar`、`AwDataTable`、`AwResourceTree`
- 详情页：`AwDetailPage`、`AwDetailToolbar`、`AwDetailHeader`、`AwDetailInfoGrid`、`AwDetailTabs`
- 新增/编辑页：`AwFormPage`、`AwEditableSubTable`、`AwAttachmentTable`、`AwRichTextEditor`、`AwSourcePickerModal`
- 设置页：`AwSettingPage`、`AwSettingSplitPage`、`AwSettingTree`、`AwSettingListCard`、`AwSettingTable`、`AwSettingModal`
- 设置能力：`AwFieldSettingPage`、`AwCodeRuleBuilder`、`AwApprovalRuleEditor`、`AwPersonPickerModal`

禁止：

- 自己写一套私有列表页。
- 自己写一套私有详情页。
- 自己写一套私有弹窗。
- 自己写一套和客户管理/销售中心不一致的设置页。
- 为了赶进度做不可编辑的静态卡片。

## 不能改的东西

没有用户明确允许，不得修改：

- 顶部一级导航顺序。
- 当前模块已确认的二级导航名称。
- 当前模块已确认的三级/四级浮窗入口。
- 已验收的客户管理母版页面。
- 公共母版组件的核心行为。
- 其他模块已验收页面。
- 接口契约里的字段语义。

如果确实需要改公共组件，必须说明：

- 为什么非改不可。
- 会影响哪些模块。
- 如何验证不破坏已有页面。

## 字段和数据来源规则

字段来源优先级：

1. 用户明确指出的字段和页面规则。
2. 已完成并通过验收的 Vue 页面。
3. 当前模块已有 Vue 页面、mock、types、resources。
4. 接口契约文件。
5. 已确认交接文档和协同公告板记录。
6. 常见业务系统经验。

常见业务系统经验只能用于补空白，不能覆盖前五项。

任何字段不确定时，必须问用户。

## 页面实现标准

### 列表页

必须满足：

- 搜索框在左侧。
- 刷新、筛选、字段配置、导入、导出、新增按钮在右侧。
- 刷新数据按钮无外框或按母版统一样式。
- 按钮必须有统一图标，除非用户明确要求不要图标。
- 选择框、序号固定在左侧。
- 操作列固定在右侧。
- 底部显示总数、已选数量、批量操作、分页。
- 全选能选中全部当前页数据。
- 表格横向滚动时固定列不能丢。
- 列表字段必须按原 JSX/契约，不得只留序号和操作。

### 详情页

必须满足：

- 顶部有返回按钮和右侧操作按钮容器。
- 基础信息是一个独立容器。
- Tab 是一个独立容器。
- Tab 可以点击切换，并显示不同内容。
- 来源单据、关联单据可点击或至少有明确入口。
- 操作按钮按用户确认文案，不得随意增删。

### 新增/编辑页

必须满足：

- 使用 `AwFormPage`。
- 子表使用 `AwEditableSubTable`。
- 附件使用统一附件组件。
- 富文本使用统一富文本组件。
- 人员/来源/产品/客户/供应商选择用弹窗，不要纯文本假装。
- 新增行、删除行、保存、取消都要有前端状态反馈。
- 保存后数据至少在当前前端状态里能看到变化。

### 设置页

必须满足：

- 自定义字段、编号、审批、策略、打印模板必须沿用客户管理/销售设置母版。
- 分类类设置使用左右结构。
- 状态使用 iOS 开关。
- 新增、编辑使用统一弹窗。
- 设置项不能写死不可编辑。
- 自定义字段弹窗、编号规则、审批流程编辑器必须和母版一致。

## 交互规则

所有按钮必须有行为：

- 新增：打开表单或弹窗。
- 编辑：进入可编辑状态或打开编辑弹窗。
- 删除：必须二次确认。
- 保存：更新前端状态并提示。
- 取消：关闭弹窗或返回列表。
- 刷新：有刷新反馈。
- 筛选：能打开筛选区域/弹窗。
- 字段配置：能打开字段配置。
- 导入/导出：可以先做模拟反馈，但不能无反应。

所有 Tab 必须能切换。

所有选择弹窗必须能选择、确认、回填。

## 业务边界规则

不要跨模块乱联动。

允许前端展示关联：

- 销售关联财务应收、开票、回款。
- 采购关联财务应付、付款、到票、三单匹配。
- 售后关联退款、应收调整、发票红冲。
- 仓储关联库存、入库、出库、盘点。
- 设备关联点检、保养、维修、备件。
- 质检关联检验、异常、返工、放行。

不允许擅自真实创建跨模块单据，除非用户或交接文档明确要求。  
如果要从“展示关联”升级为“真实联动”，必须先问。

## 遇到不确定必须问

以下情况不能自己决定：

- 用户说“不要用原来的”或“按现在的”但范围不清楚。
- 当前 Vue、交接文档、接口契约和用户最新意见不一致。
- 要新增、删除、改名导航。
- 要改公共母版组件。
- 要删除字段。
- 要合并页面。
- 要把设置页简化成静态展示。
- 要做跨模块真实联动。
- 字段有乱码且无法从上下文确定中文。

提问格式：

- 先说明卡在哪里。
- 给出 1 个推荐处理方式。
- 不要一次问太多。

## 禁止行为

禁止：

- 只凭模块名称生成通用页面。
- 用“差不多”字段替代原字段。
- 没看母版就写页面。
- 没看当前 Vue 页面、母版和验收意见就写列表字段。
- 把列表页做成只有序号和操作。
- 把详情 Tab 做成不能切换。
- 把设置页写成静态卡片。
- 擅自删掉用户点名的按钮、字段、状态。
- 擅自更改已验收模块。
- 把问题留给用户反复传话。

## 自检清单

每个窗口交付前必须自检并在回复里列出来：

1. 读过哪些文件。
2. 对应路由有哪些。
3. 列表页是否字段完整、固定列正常、批量选择正常。
4. 详情页 Tab 是否能切换。
5. 新增/编辑页是否能保存并回显。
6. 弹窗是否能打开、选择、确认、回填。
7. 设置页是否复用母版。
8. 是否改过公共组件。
9. 是否影响其他模块。
10. `npm run build` 是否通过；如果没跑，说明原因。

## 新窗口统一复制指令

你接下来负责一个 ERP 模块迁移/修复。开工前必须先读：

- `docs/new-window-general-rules.md`
- `vue3/src/views/template-gallery/TemplateGallery.vue`
- `vue3/src/layouts/erp-shell/navigation.ts`
- `vue3/src/app/router/index.ts`
- 当前模块的交接文档、验收记录、Vue 页面、mock、types、resources、接口契约

执行要求：

- 先看母版，不允许自己写一套私有页面。
- 字段和交互以用户最新确认、已验收 Vue、当前模块现有实现、接口契约为准。
- 当前已进入验收调整阶段，不再要求参考原 JSX 页面。
- 不确定必须问，不允许自己发挥。
- 列表、详情、新增、设置、弹窗都要能真实交互。
- 交付前必须按自检清单检查，并说明改了哪些文件、哪些路由可验收、哪些问题需要用户确认。
