# 售后中心协调文档

## 当前交付范围

- 售后主单资源：`after-sales/services`
- 售后执行任务：`after-sales/tasks`
- 关联单据状态：`after-sales/linked-documents`
- 质量闭环：`after-sales/quality-actions`
- 第一阶段设置：售后原因、投诉问题、售后类型、处理方式

## 已确认规则

- 质量闭环不阻塞售后结单；售后结单后仍可显示“已关闭/质量追踪中”。
- “客户确认”调整为“结单确认”，由内部人员录入作为结单动作。
- 应收调整、发票红冲第一阶段先作为关联单据展示，并参与退款类结单条件。
- 配件出库单第一阶段并入出库单状态机。

## 状态机

- 入库单：待入库 -> 已入库 -> 已质检
- 出库单/配件出库单：待出库 -> 已出库 -> 已签收
- 付款单：待审核 -> 待付款 -> 已付款
- 服务派工：待派工 -> 服务中 -> 已完成
- 应收调整：待调整 -> 已调整
- 发票红冲：待红冲 -> 已红冲
- 质量闭环：待分析 -> CAPA执行中 -> 已验证 -> 已关闭

## 回填规则

- 退款退货：退货入库单已质检 + 退款付款单已付款 + 应收调整已调整 + 发票红冲已红冲 -> 待结单确认。
- 仅退款：退款付款单已付款 + 发票红冲已红冲 -> 待结单确认。
- 换货：退货入库单已质检 + 换货出库单已签收 -> 待结单确认。
- 仅退货：退货入库单已质检 + 应收调整已调整 -> 待结单确认。
- 维修/现场服务：服务派工单已完成 + 配件出库单已签收 -> 待结单确认。

## 页面入口

- 工作台：`/after-sales`
- 售后单列表：`/after-sales/services`
- 新增售后：`/after-sales/services?action=new`
- 售后详情：`/after-sales/services?id=as_001`
- 任务管理：`/after-sales/tasks`
- 任务详情：`/after-sales/tasks?id=task_doc_001`
- 质量闭环：`/after-sales/quality`
- 质量详情：`/after-sales/quality?id=qa_001`
- 售后设置：`/after-sales/settings`

## 验收镜子

- 静态验收镜子：`vue3/preview-after-sales.html`
- 本地 Vue 服务：`http://127.0.0.1:5177`

验收项：

- 工作台卡片可跳转到售后列表、任务列表、质量闭环和设置页。
- 售后列表字段、分类树、筛选、操作列符合主单入口要求。
- 新增售后可选择客户、来源单据、回填数量金额，并预览派生单据。
- 提交后自动生成退货入库、退款付款、换货出库、服务派工、配件出库、应收调整、发票红冲等关联单据。
- 售后详情的关联单据可推进状态，并反写主单汇总状态。
- 任务列表和详情从派生单据生成，推进任务会同步推进派生单据状态。
- 质量闭环可由售后详情升级生成，质量追踪不阻塞售后结单。
- 设置页第一阶段只维护四类字典，复杂编号、审批、策略、打印模板后续接公共设置组件。
