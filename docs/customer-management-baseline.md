# Customer Management Baseline

本文档用于冻结当前客户管理样板，作为后续提取公共组件、制作母版和复用到销售中心其他类目的基准。本文只描述当前静态预览与共享样式中的既定结构，不覆盖接口契约，也不要求立即抽象实现。

## 1. 样板页面清单

| 页面 | 文件 | 样板职责 |
| --- | --- | --- |
| 销售工作台 | `vue3/preview-framework.html` | 销售中心默认工作台，包含顶部一级导航、左侧二级导航、客户管理浮窗、待办事项、业务导航、便捷入口、公告和最近访问。 |
| 客户列表 | `vue3/preview-customer-list.html` | 客户档案标准列表页，包含左侧分类树、搜索和工具栏、横向可滚动表格、固定列、批量操作和分页。 |
| 客户详情 | `vue3/preview-customer-detail.html` | 客户档案详情页，包含返回栏、右侧操作按钮、客户摘要信息、Tab 容器、基础信息和额度指标。 |
| 新增客户 | `vue3/preview-customer-new.html` | 新增/编辑表单页，包含基础信息、客户信息 Tab 表格、账期设置、富文本客户详情。 |
| 客户设置 | `vue3/preview-customer-setting.html` | 客户设置集合页，覆盖客户分组、自定义字段、自定义编号、等级、审批、策略等二级设置页面。 |
| 共享样式 | `vue3/src/styles/aw-console.css` | 当前客户管理静态样板的主要布局、表格、卡片、表单、弹窗、线性图标和状态样式来源。 |

## 2. 每页冻结关键点

### 2.1 销售工作台

必须冻结的结构：

- 顶部一级导航使用 `.aw-topbar`，销售导航项保持高亮并链接到销售工作台。
- 主框架使用 `.aw-shell`，左侧固定宽度导航 `.aw-side`，右侧内容区 `.aw-main`。
- 左侧“工作台”作为销售中心默认入口；“客户管理”悬浮出现三级/四级菜单。
- 客户管理浮窗分为“客户管理”和“客户设置”两段，包含添加客户、客户列表、客户分组设置、自定义字段、自定义编号、等级、审批、策略入口。
- 主内容顺序保持为：页面标题、待办事项、业务导航、便捷入口、公告/消息中心、最近访问。

必须冻结的交互：

- 鼠标移入客户管理二级导航显示浮窗，移出后隐藏。
- 待办事项默认一行七个卡片；“全部”打开待办列表弹窗，“展开”显示全部待办。
- “刷新”触发刷新态，刷新时图标/文字显示绿色刷新状态，完成后恢复。
- 待办事项、业务导航、便捷入口内卡片允许拖动调整顺序。
- 公告/消息中心和最近访问最多展示 5 条，二者条目行高保持一致。

必须冻结的视觉：

- 卡片圆角统一 8px，页面背景为浅灰，内容卡片为白底细边框。
- 工作台待办卡片使用淡色块加虚线边框，数字使用较大字重。
- 业务导航卡片使用角标数字、线性/轻量图标、统一宫格。
- 全局图标原则为线条型，不使用实心 emoji 作为最终图标母版。

### 2.2 客户列表

必须冻结的结构：

- 页面主结构为 `.aw-list-layout`：左侧 `.aw-doc-tree`，右侧 `.aw-list-panel`。
- 左侧只保留“客户库 (4)”标题和“客户分组”树；不再展示单独的“全部客户”条目。
- 客户库后的数量与标题字号一致，并使用新增客户按钮同色。
- 工具栏左侧为全局搜索；右侧依次为刷新数据、筛选、字段配置、导入、导出、新增客户。
- 表格包含选择框、序号、客户名称、客户分组、主联系人、职位、联系方式、信用、操作等字段。
- 底部包含全选、共几条、已选几条、批量转移、批量指定和分页。

必须冻结的交互：

- 左侧客户分组点击后筛选右侧客户列表。
- 表头全选可以选中全部行；底部已选数量同步变化。
- 横向滚动时选择框、序号固定在左侧，操作列固定在右侧。
- “新增客户”跳转到新增客户页。
- 客户列表入口可从客户管理浮窗进入。

必须冻结的视觉：

- 左侧树图标使用线条型；客户分组使用人员类线性图标，下级分类使用统一线性节点图标。
- 刷新数据为轻量文字按钮，包含刷新图标，无需强化外框。
- 批量操作按钮紧跟已选文字后展示。

### 2.3 客户详情

必须冻结的结构：

- 页面主结构为 `.aw-detail-page`。
- 第一块为返回和操作按钮容器 `.aw-detail-toolbar`。
- 第二块为客户基本信息摘要 `.aw-detail-header`。
- 第三块为 Tab 内容容器，包含客户信息、产品记录、购买记录、发货记录、开票记录、回款核销、售后记录、操作记录。
- 基础信息下方展示额度指标卡片。

必须冻结的交互：

- 返回客户列表按钮返回客户列表页。
- 右侧操作按钮为：修改、删除、打印、导出、停用。
- 停用按钮使用红色边框和红色文字。
- 右侧操作按钮不带图标。

必须冻结的视觉：

- 返回按钮使用箭头，文字颜色与按钮一致。
- 客户摘要里客户名称、审核状态、编号、客户分组、主联系人、联系方式、客户经理排列在同一个信息区内。
- “客户分组：”“主联系人：”“联系方式：”“客户经理：”等标签和值样式一致。
- Tab 位于独立容器内，选中态为蓝色文字和底部蓝线。

### 2.4 新增客户

必须冻结的结构：

- 页面主结构为 `.aw-form-page`，各区块为 `.aw-form-card`。
- 基础信息区为三列表单，包含客户名称、客户分类、客户经理、拼音码、信用代码、客户编号、客户等级、所属行业、信用管控选择项。
- “信用管控”字段不显示单独标签。
- 客户信息区使用 Tab：联系人信息、财务信息、地址信息、附件信息。
- 四个 Tab 都使用表格形式，并支持新增一行。
- 账期设置在客户信息区内，位于 Tab 表格下方。
- 客户详情为富文本编辑器样式，而不是普通 textarea。
- 独立附件区已取消；附件信息归入客户信息 Tab。

必须冻结的交互：

- 新增联系人/财务/地址/附件点击后新增可编辑行。
- 账期设置卡片支持单选切换。
- 现结、月结、周期、额度的输入框在各自卡片内部。
- 现结默认文案为“下单时即时结清货款”。
- 月结输入示例为“如每个月25号结算”。
- 周期输入示例为“如30、60、90按照指定天数结算”。
- 额度输入示例为“如200000”，并提示“超出额度需提前收款或审批”。

必须冻结的视觉：

- 表单输入框、选择框、按钮、Tab 和表格样式与客户列表/详情保持一致。
- 账期卡片选中态为蓝色边框和浅蓝背景。
- 富文本工具栏保留文件、编辑、插入、视图、格式、表格，以及常用格式按钮。

### 2.5 客户设置

必须冻结的结构：

- 页面主结构为 `.aw-setting-page`，顶部为返回栏和操作按钮容器。
- 客户分组、自定义字段使用左右结构 `.aw-setting-split`。
- 自定义编号使用左右双卡片结构 `.aw-code-builder`。
- 等级设置使用普通列表卡片。
- 审批设置进入编辑态后使用审批规则设置表单。
- 策略设置使用 Tab + 策略行配置卡片。

客户分组设置：

- 左侧标题为“客户分类”，右侧有加号按钮用于新增一级分类。
- 一级分类弹窗字段为分类名称、分类编号、分类备注。
- 右侧标题显示当前一级分类名称；标题下方显示该一级分类备注，必须使用新增分类时填写的备注。
- 右侧列表显示当前一级分类下级分类。
- 列表不显示默认属性字段。

客户自定义字段：

- 左侧为字段位置：基础信息、客户信息、财务信息、地址信息、附件信息、客户详情。
- 右侧显示当前字段位置下的自定义字段。
- 新增字段弹窗包含字段名称、字段编码、字段类型、是否必填、默认值、提示文字、权限设置。
- 权限设置通过选择负责人弹窗添加人员。
- 权限人员列表为表格形式，表头保留姓名、部门、职位、是否可见；不保留只读。
- 字段列表需要包含是否必填、是否启用，均使用 iOS 开关按钮；操作只保留编辑、删除。

客户自定义编号：

- 左侧展示当前样式预览和已选编号项。
- 固定前缀必须存在，占 1 个名额。
- 新增编号项与固定前缀在同一行组合预览。
- 固定前缀为黑色文字，新增编号项使用系统主按钮同色。
- 右侧配置规则名称、规则编码、间隔符、编号项、扩展项。
- 编号项最多 5 个，前缀固定占 1 个，其余最多选择 4 项。

客户等级设置：

- 列表字段为序号、等级名称、备注、是否启用、操作。
- 不显示折扣、授信参考、判定条件。
- 是否启用使用 iOS 开关按钮。
- 操作只保留编辑、删除，不显示停用。
- 新增等级使用弹窗，字段为等级名称、是否启用、备注。

客户审批设置：

- 编辑界面标题为“审批规则设置”。
- 顶部右侧按钮为取消、编辑、保存，其中保存为蓝色主按钮。
- 底部不再重复显示取消和保存。
- 流程基础信息只保留流程名称。
- 不显示“适用分类”字段和下拉框。
- 审批节点为横向卡片流，节点之间用箭头连接。
- 节点内包含节点名称、审批人、审批方式。
- 审批方式支持依次审批、会签、或签，并都必须可选中。
- 审批人可多选，再次选择只增加人员，不自动删除已选人员；删除只能点击已选人员标签上的删除。
- 审批人输入框下方展示已选人员标签。

客户策略设置：

- 删除客户等级规则。
- 删除所有“流程来源：客户审批设置。”字样。
- 右侧审批下拉选项来源为客户审批设置列表。
- 保留客户建档、财务与默认信息、审批规则三个 Tab。
- 策略行由标题、说明和右侧控件组成；控件可以是审批流程下拉或启用/关闭单选。

## 3. 可作为母版参数的内容

后续提取母版时，以下内容建议做成配置或接口数据，而不是写死在组件内部：

- 模块基础信息：模块 key、模块名称、一级导航名称、左侧二级导航菜单。
- 菜单结构：二级菜单、三级/四级浮窗分组、菜单项名称、路由、权限码。
- 工作台卡片：待办事项、业务导航、便捷入口的标题、数量、颜色、图标、跳转地址、是否可拖拽、排序值。
- 消息列表：公告/消息中心和最近访问的数据源、条数限制、过滤标签。
- 列表页配置：左侧分类树标题、分类数据、搜索 placeholder、工具栏按钮、表格列、固定列、批量操作、分页。
- 详情页配置：摘要字段、状态字段、Tab 列表、指标卡片、操作按钮。
- 新增/编辑页配置：基础表单字段、Tab 表格 schema、账期选项、富文本字段、附件字段。
- 设置页类型：左右结构设置、普通列表设置、编号规则设置、审批流程设置、策略设置。
- 设置页数据：分类树、字段位置、编号候选项、审批流程列表、策略规则列表。
- 弹窗配置：新增分类、字段、等级、人员选择器等弹窗标题、字段 schema、提交行为。
- 视觉 token：主色、危险色、成功色、边框、圆角、间距、表格行高、按钮高度、线性图标映射。

## 4. 暂不抽象或高风险点

以下内容暂时不建议在第一轮强行抽象，避免母版过早复杂化：

- 审批流程编辑器：节点横向编排、多选审批人、会签/或签/依次审批、人员弹窗回填逻辑较多，建议先单独封装再接入设置母版。
- 自定义编号规则：前缀固定、候选项互斥、拖拽排序、预览拼接、搜索项开关等逻辑独立性强，不宜塞进普通设置表格母版。
- 富文本编辑器：当前是样式复刻，后续接入真实编辑器时会带来第三方库和内容序列化问题。
- 表格固定列：当前依赖 sticky 样式和横向滚动容器，抽成通用表格前需要验证宽度、阴影、表头和 hover 背景。
- 拖拽排序：工作台三类卡片都有拖拽诉求，但保存顺序、拖拽占位、跨容器拖拽规则还未定义。
- 附件上传：新增客户页已确定附件信息走 Tab 表格，但真实上传、查看、下载、文件对象 URL 和接口返回格式需要单独契约。
- 人员选择器：可复用价值高，但会涉及部门树、已选回显、跨业务角色过滤和权限控制，建议作为独立公共组件。
- 客户设置多个子页共用一个静态 HTML 和脚本，抽象前需要先明确 Vue 中是按路由拆页面，还是按 `settingType` 配置渲染。
- 全局图标替换：目前部分静态区域仍残留文本符号或 emoji，抽象母版时应统一映射为线条型图标，不能简单照搬字符。
- 接口契约接入：本文只冻结页面样板，接口字段、分页、排序、上传、审批保存等仍需按契约单独确认。

## 5. 冻结结论

客户管理当前样板可以作为销售中心后续模块的第一套母版来源。建议先按页面类型提取：工作台母版、列表页母版、详情页母版、新增/编辑页母版、设置页母版；再用客户管理自身回填验证，确认视觉和交互不回退后，再复用到销售计划、报价、合同、订单等模块。
