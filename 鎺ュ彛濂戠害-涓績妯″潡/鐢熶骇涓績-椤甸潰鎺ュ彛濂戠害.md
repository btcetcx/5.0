# 生产中心页面接口契约

适用模块：生产中心  
接口前缀：`/api/v1`  
原型参考：`manufacturing-list.jsx`

## 1. 模块范围

| 页面 | 原型文件 | 当前完整度 | 主要能力 |
| --- | --- | --- | --- |
| 生产需求 | `manufacturing-list.jsx` | 完整 | 新增需求、需求列表、需求汇总、从销售订单/手动来源生成需求 |
| 生产计划 | `manufacturing-list.jsx` | 完整 | 新增计划、计划列表、产品明细、齐套预估、生成生产订单 |
| 生产订单 | `manufacturing-list.jsx` | 完整 | 新增订单、订单列表、单产品订单、拆解工单 |
| 生产工单 | `manufacturing-list.jsx` | 完整 | 新增工单、工单列表、领工派工、任务报工、报工记录 |
| 生产排班 | `manufacturing-list.jsx` | 完整 | 排班列表、排班计划、班组、工作日历、班次管理 |
| 委外加工 | `manufacturing-list.jsx` | 完整 | 委外加工单、委外发料、委外入库 |

## 2. 通用枚举

### 来源类型

| sourceType | sourceTypeName |
| --- | --- |
| `salesOrder` | 销售订单 |
| `stockReplenish` | 库存备货 |
| `manual` | 手动创建 |
| `productionDemand` | 生产需求 |
| `productionPlan` | 生产计划 |
| `productionOrder` | 生产订单 |

### 生产状态

| status | statusName | 适用 |
| --- | --- | --- |
| `draft` | 待提交 | 通用 |
| `pendingConfirm` | 待确认 | 生产需求 |
| `pendingApproval` | 待审批 | 生产计划、订单、工单 |
| `approved` | 已审批 | 通用 |
| `planning` | 排产中 | 生产计划 |
| `pendingProduction` | 待生产 | 生产订单 |
| `producing` | 生产中 | 订单、工单 |
| `pendingReport` | 待报工 | 工单 |
| `reported` | 已报工 | 工单 |
| `completed` | 已完成 | 通用 |
| `delayed` | 已延期 | 计划、订单 |
| `closed` | 已关闭 | 通用 |
| `rework` | 返工 | 订单、工单 |

### 优先级

| priority | priorityName |
| --- | --- |
| `normal` | 普通 |
| `urgent` | 紧急 |
| `critical` | 特急 |

## 3. 生产需求

### 页面字段

基础信息：需求主题、需求编号、生产来源、销售订单、客户名称、订单交付日期、生产项目、目标仓库、计划类型、来源主体、来源交付日期、需求状态、生产部门、负责人、计划开始日期、计划完成日期、优先级。

产品明细：来源类型、来源单据、来源行号、产品编码、产品名称、规格型号、单位、需求数量、计划数量、BOM版本、工艺路线、计划开始、计划完成。

### 接口

```text
GET    /api/v1/production-demands
POST   /api/v1/production-demands
GET    /api/v1/production-demands/{id}
PATCH  /api/v1/production-demands/{id}
POST   /api/v1/production-demands/{id}/submit
POST   /api/v1/production-demands/{id}/confirm
POST   /api/v1/production-demands/{id}/close
POST   /api/v1/production-demands/{id}/generate-plan
POST   /api/v1/production-demands/{id}/generate-order
GET    /api/v1/production-demand-sources
```

## 10. 需要和后端确认

- 生产需求是否必须来自销售订单，还是允许库存备货/手动创建。
- 一个生产订单是否严格只对应一个生产产品。
- 生产订单拆工单由前端触发，还是后端按 BOM/工艺自动拆解。
- 领工、派工、报工是否需要独立审批。
- 报工后是否自动生成质检任务、成品入库通知。
- 委外发料是否调用仓储出库，委外入库是否调用仓储入库。
- 排班冲突校验由前端提示还是后端阻断发布。
