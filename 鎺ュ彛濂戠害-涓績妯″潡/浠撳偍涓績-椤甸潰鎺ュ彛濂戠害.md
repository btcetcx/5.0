# 仓储中心页面接口契约

适用模块：仓储中心  
接口前缀：`/api/v1`  
原型参考：`warehouse-stock-list.jsx`、`warehouse-inbound-list.jsx`、`warehouse-outbound-list.jsx`、`warehouse-transfer-list.jsx`、`warehouse-inventory-list.jsx`

## 1. 模块范围

| 页面 | 原型文件 | 当前完整度 | 主要能力 |
| --- | --- | --- | --- |
| 库存管理 | `warehouse-stock-list.jsx` | 完整 | 库存列表、库存明细、库存调整、库存流水 |
| 入库管理 | `warehouse-inbound-list.jsx` | 完整 | 直接入库、采购入库、生产入库、销售退货入库、委外入库 |
| 出库管理 | `warehouse-outbound-list.jsx` | 完整 | 直接出库、内部领用、委外领料、销售出库、采购退货出库 |
| 调拨管理 | `warehouse-transfer-list.jsx` | 完整 | 新增调拨、调拨列表、调拨明细 |
| 盘点管理 | `warehouse-inventory-list.jsx` | 完整 | 直接盘点、盘点计划、盘点单、差异调整 |
| 出库质检 | 导航预留 | 预留 | 出库质检任务 |
| 来料质检 | 导航预留 | 预留 | 来料质检任务 |
| 仓库库位 | 仓储库位屏 | 部分 | 仓库、库区、库位维护 |

## 2. 通用枚举

### 入库类型

| inboundType | inboundTypeName |
| --- | --- |
| `direct` | 直接入库 |
| `purchase` | 采购入库 |
| `production` | 生产入库 |
| `salesReturn` | 销售退货入库 |
| `outsource` | 委外入库 |

### 出库类型

| outboundType | outboundTypeName |
| --- | --- |
| `direct` | 直接出库 |
| `internalUse` | 内部领用 |
| `outsourceIssue` | 委外领料 |
| `sales` | 销售出库 |
| `purchaseReturn` | 采购退货 |

### 仓储状态

| status | statusName | 适用 |
| --- | --- | --- |
| `draft` | 草稿 | 入库、出库、调拨、盘点 |
| `pendingReview` | 待复核 | 出库、入库 |
| `receivedPendingQc` | 已收货待检 | 入库 |
| `qualifiedPendingShelf` | 合格待上架 | 入库 |
| `partialShelved` | 部分上架 | 入库 |
| `posted` | 已上架过账/已出库过账 | 入库、出库 |
| `pendingOccupy` | 已审核待占用 | 出库 |
| `shortPickPending` | 短拣待处理 | 出库 |
| `notStarted` | 未开始 | 盘点 |
| `counting` | 盘点中 | 盘点 |
| `rechecking` | 复盘中 | 盘点 |
| `differencePendingAdjust` | 差异待调整 | 盘点 |
| `closed` | 关闭 | 通用 |

```text
GET /api/v1/warehouse/settings/code-rules?module={module}
PUT /api/v1/warehouse/settings/code-rules?module={module}
GET /api/v1/warehouse/settings/custom-fields?module={module}
PUT /api/v1/warehouse/settings/custom-fields?module={module}
GET /api/v1/warehouse/settings/approval-flows?module={module}
PUT /api/v1/warehouse/settings/approval-flows?module={module}
GET /api/v1/warehouse/settings/policies?module={module}
PUT /api/v1/warehouse/settings/policies?module={module}
GET /api/v1/warehouse/settings/print-templates?module={module}
PUT /api/v1/warehouse/settings/print-templates?module={module}
```

`module` 可选：`stock`、`inbound`、`outbound`、`transfer`、`inventoryCount`、`warehouseLocation`。

## 11. 需要和后端确认

- 入库过账是否必须完成质检，哪些入库类型可跳过质检。
- 出库是否先占用库存再拣货，还是提交后直接扣减。
- 序列号、批次号、箱码是否统一建模为库存标识。
- 盘点锁库是按仓库、库位、批次还是物料维度锁定。
- 销售出库、采购入库、生产入库是否由对应模块自动生成待处理通知。
- 库存流水是否作为所有库存变化的唯一审计来源。
