# 销售中心页面接口契约

适用模块：销售中心  
接口前缀：`/api/v1`  
原型参考：`customer-list.jsx`、`sales-plan-list.jsx`、`quote-list.jsx`、`contract-list.jsx`、`sale-order-list.jsx`

## 1. 模块范围

| 页面 | 原型文件 | 当前完整度 | 主要能力 |
| --- | --- | --- | --- |
| 客户管理 | `customer-list.jsx` | 完整 | 客户列表、新增、详情、联系人、地址、开票、信用额度 |
| 销售计划 | `sales-plan-list.jsx` | 完整 | 计划列表、新增、详情、计划产品、完成情况 |
| 报价管理 | `quote-list.jsx` | 完整 | 报价列表、新增、详情、阶梯报价、转合同/订单 |
| 合同管理 | `contract-list.jsx` | 完整 | 合同列表、新增、详情、合同产品、收款/开票/执行 |
| 订单管理 | `sale-order-list.jsx` | 完整 | 订单列表、新增、详情、来源选择、信用校验、发货进展 |
| 销售退货 | 导航预留 | 预留 | 退货列表、新增退货、退货明细 |
| 销售换货 | 导航预留 | 预留 | 换货列表、新增换货、换货明细 |
| 销售报表 | 导航预留 | 预留 | 销售对账、按客户查询、按产品统计、退换货明细 |

## 2. 通用枚举

### 客户状态

| status | statusName |
| --- | --- |
| `pendingApproval` | 待审核 |
| `approved` | 已审核 |
| `following` | 跟进中 |
| `disabled` | 已停用 |

### 信用状态

| creditStatus | creditStatusName |
| --- | --- |
| `normal` | 正常 |
| `nearLimit` | 临近额度 |
| `blocked` | 信用拦截 |
| `cashOnly` | 现结 |
| `disabled` | 停止赊销 |

### 销售单据状态

| status | statusName | 适用 |
| --- | --- | --- |
| `draft` | 草稿 | 报价、合同、订单 |
| `pendingApproval` | 审核中 | 报价、合同、订单 |
| `approved` | 已批准 | 报价、合同、订单 |
| `confirmed` | 已确认 | 订单 |
| `executing` | 执行中 | 计划、合同 |
| `completed` | 已完成 | 通用 |
| `paused` | 已暂停 | 计划 |
| `closed` | 已关闭 | 通用 |
| `expired` | 已失效 | 报价 |
| `cancelled` | 已取消 | 订单 |

### 订单进展

| progressStatus | progressName |
| --- | --- |
| `notShipped` | 未发货 |
| `partialShipped` | 部分发货 |
| `shipping` | 发货中 |
| `shipped` | 已发货 |
| `producing` | 生产中 |
| `completed` | 已完成 |

## 3. 客户管理

### 页面字段

列表字段：客户名称、客户分组、主联系人、职位、联系方式、信用额度、已用额度、占用额度、应收未收、可用额度、账期、信用状态、客户经理。

新增/详情字段：

| 分组 | 字段 |
| --- | --- |
| 基础信息 | 客户名称、客户分类、客户经理、拼音码、信用代码、客户编号、客户等级、所属行业、信用管控 |
| 联系人 | 联系人、职位、手机号、邮箱、是否默认、备注 |
| 地址 | 地址类型、收件人、联系电话、详细地址、是否默认 |
| 开票信息 | 发票抬头、税号、开户行、银行账号、注册地址、电话 |
| 信用信息 | 信用额度、已用额度、占用额度、应收未收、可用额度、账期、风险状态 |

GET    /api/v1/customers

POST   /api/v1/customers/{id}/reject
POST   /api/v1/customers/{id}/disable
GET    /api/v1/customer-groups/tree

```text

PUT /api/v1/sales/settings/code-rules?module={module}
GET /api/v1/sales/settings/custom-fields?module={module}
PUT /api/v1/sales/settings/custom-fields?module={module}
GET /api/v1/sales/settings/approval-flows?module={module}
PUT /api/v1/sales/settings/approval-flows?module={module}
GET /api/v1/sales/settings/policies?module={module}
PUT /api/v1/sales/settings/policies?module={module}
GET /api/v1/sales/settings/print-templates?module={module}
PUT /api/v1/sales/settings/print-templates?module={module}
```

`module` 可选：`customer`、`salesPlan`、`quote`、`contract`、`salesOrder`、`salesReturn`、`salesExchange`。

## 10. 需要和后端确认

- 客户信用额度是否由销售模块维护，还是由财务应收模块回写。
- 报价转合同、报价转订单是否允许多次转化。
- 合同是否必须存在，订单是否支持“非必须合同”。
- 销售订单释放出库是自动生成出库单，还是需要仓储人工确认。
- 开票申请与回款是否由财务模块回写到销售订单。
