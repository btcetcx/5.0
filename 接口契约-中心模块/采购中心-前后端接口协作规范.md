# 采购中心前后端接口协作规范

适用模块：采购中心  
覆盖页面：供应商管理、请购管理、询价管理、采购订单、采购退货、采购换货、采购报表  
接口前缀：`/api/v1`

> 说明：本文件定义采购中心前后端协作的通用约束；具体字段、枚举、接口路径见《采购中心-页面接口契约.md》。

## 1. 协作目标

- 前端 mock 数据、页面字段、API adapter 与后端接口保持同一套字段口径。
- 后端可以按本规范先返回 mock 同结构数据，再逐步接入真实业务逻辑。
- 联调验收以前端页面契约为准，接口命名、分页、状态、错误响应按本规范执行。

## 2. 基础约定

| 项 | 约定 |
| --- | --- |
| API 前缀 | `/api/v1` |
| 数据格式 | `application/json` |
| 字符编码 | `UTF-8` |
| 日期 | `YYYY-MM-DD` |
| 日期时间 | ISO 8601，例如 `2026-06-07T10:12:00+08:00` |
| 字段命名 | `camelCase` |
| 主键 | `id` |
| 业务编号 | `code` |
| 金额 | number，币种单独返回 `currency` |
| 数量 | number，单位单独返回 `unitName` |

## 3. 采购中心资源命名

| 页面 | 推荐资源路径 | 说明 |
| --- | --- | --- |
| 供应商管理 | `/suppliers` | 供应商档案、联系人、地址、财务、供货产品 |
| 请购管理 | `/purchase-requests` | 请购单、请购明细、转询价、转采购 |
| 询价管理 | `/purchase-inquiries` | 询价单、供应商报价、定价、临时供应商 |
| 采购订单 | `/purchase-orders` | 采购订单、到货、入库、付款、三单匹配 |
| 采购退货 | `/purchase-returns` | 预留，退货单与退货明细 |
| 采购换货 | `/purchase-exchanges` | 预留，换货单与换货明细 |
| 采购报表 | `/purchase-reports` | 预留，供应商对账、按产品统计 |

## 4. 统一响应结构

成功响应：

```json
{
  "success": true,
  "data": {},
  "message": ""
}
```

列表响应：

```json
{
  "success": true,
  "data": {
    "items": [],
    "page": {
      "pageNo": 1,
      "pageSize": 20,
      "total": 156,
      "pages": 8
    }
  },
  "message": ""
}
```

错误响应：

```json
{
  "success": false,
  "code": "PURCHASE_ORDER_NOT_FOUND",
  "message": "采购订单不存在",
  "details": {}
}
```

## 5. 列表查询约定

通用 Query 参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `pageNo` | number | 页码，从 1 开始 |
| `pageSize` | number | 每页条数 |
| `keyword` | string | 全局搜索 |
| `status` | string | 状态过滤，多个值用逗号分隔 |
| `supplierId` | string | 供应商过滤 |
| `materialId` | string | 物料过滤 |
| `ownerId` | string | 负责人过滤 |
| `createdFrom` | string | 创建日期起 |
| `createdTo` | string | 创建日期止 |
| `sortBy` | string | 排序字段 |
| `sortOrder` | string | `asc` 或 `desc` |

## 6. 状态与动作约定

- 状态判断统一使用 `status`，页面展示使用 `statusName`。
- 前端不要用中文状态判断业务逻辑。
- 审批类动作统一使用：

```text
POST /api/v1/{resource}/{id}/submit
POST /api/v1/{resource}/{id}/approve
POST /api/v1/{resource}/{id}/reject
POST /api/v1/{resource}/{id}/revoke
```

采购中心常用动作：

```text
POST /api/v1/purchase-requests/{id}/transfer-inquiry
POST /api/v1/purchase-requests/{id}/transfer-order
POST /api/v1/purchase-inquiries/{id}/send
POST /api/v1/purchase-inquiries/{id}/price
POST /api/v1/purchase-inquiries/{id}/create-order
POST /api/v1/purchase-orders/{id}/confirm
POST /api/v1/purchase-orders/{id}/close
POST /api/v1/purchase-orders/{id}/create-inbound
POST /api/v1/purchase-orders/{id}/create-payment
```

## 7. Mock 切换约定

- mock 字段必须覆盖页面契约中的关键字段。
- 后端首版接口可以先返回 mock 同结构数据。
- mock 到真实接口切换只改 API adapter，不改页面组件字段。
- 采购中心建议拆分 mock：

```text
mock/purchase/
  suppliers.json
  purchase-requests.json
  purchase-inquiries.json
  purchase-orders.json
  purchase-returns.json
  purchase-exchanges.json
```

## 8. 联调验收清单

- 供应商、请购、询价、采购订单列表可分页、搜索、筛选。
- 详情接口字段覆盖页面详情、明细行、操作记录。
- 新增、编辑、提交、审批、转询价、转采购、定价、确认订单等动作有明确返回状态。
- 金额、数量、税率、付款、入库字段使用 number。
- 采购退货、采购换货、采购报表如暂不开发，应返回明确的预留范围和后续版本计划。
