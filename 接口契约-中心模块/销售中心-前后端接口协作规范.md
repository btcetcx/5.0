# 销售中心前后端接口协作规范

适用模块：销售中心  
覆盖页面：客户管理、销售计划、报价管理、合同管理、订单管理、销售退货、销售换货、销售报表  
接口前缀：`/api/v1`

> 说明：本文件定义销售中心前后端协作的通用约束；具体字段、枚举、接口路径见《销售中心-页面接口契约.md》。

## 1. 协作目标

- 前端 mock 数据、页面字段、API adapter 与后端接口保持同一套字段口径。
- 后端可以按本规范先返回 mock 同结构数据，再逐步接入真实业务逻辑。
- 联调验收以前端页面契约为准，接口命名、分页、状态、错误响应按本规范执行。

## 2. 基础约定

| 项 | 约定 |
| --- | --- |
| API 前缀 | `/api/v1` |
| 数据格式 | `application/json` |
| 字符编码 | `UTF-8` |
| 日期 | `YYYY-MM-DD` |
| 日期时间 | ISO 8601，例如 `2026-06-07T10:12:00+08:00` |
| 字段命名 | `camelCase` |
| 主键 | `id` |
| 业务编号 | `code` |
| 金额 | number，币种单独返回 `currency` |

## 3. 销售中心资源命名

| 页面 | 推荐资源路径 | 说明 |
| --- | --- | --- |
| 客户管理 | `/customers` | 客户档案、联系人、地址、财务信息 |
| 销售计划 | `/sales-plans` | 销售目标、计划周期、达成率 |
| 报价管理 | `/sales-quotes` | 报价单、价格版本、转化记录 |
| 合同管理 | `/sales-contracts` | 合同、履约、订单核销、开票回款 |
| 订单管理 | `/sales-orders` | 销售订单、发货、生产、应收、回款 |
| 销售退货 | `/sales-returns` | 预留，退货单与退货明细 |
| 销售换货 | `/sales-exchanges` | 预留，换货单与换货明细 |
| 销售报表 | `/sales-reports` | 预留，按客户、产品、退换货统计 |

## 4. 统一响应结构

成功响应：

```json
{
  "success": true,
  "data": {},
  "message": ""
}
```

列表响应：

```json
{
  "success": true,
  "data": {
    "items": [],
    "page": {
      "pageNo": 1,
      "pageSize": 20,
      "total": 156,
      "pages": 8
    }
  },
  "message": ""
}
```

错误响应：

```json
{
  "success": false,
  "code": "SALES_ORDER_NOT_FOUND",
  "message": "销售订单不存在",
  "details": {}
}
```

## 5. 列表查询约定

通用 Query 参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `pageNo` | number | 页码，从 1 开始 |
| `pageSize` | number | 每页条数 |
| `keyword` | string | 全局搜索 |
| `status` | string | 状态过滤，多个值用逗号分隔 |
| `customerId` | string | 客户过滤 |
| `ownerId` | string | 销售负责人过滤 |
| `createdFrom` | string | 创建日期起 |
| `createdTo` | string | 创建日期止 |
| `sortBy` | string | 排序字段 |
| `sortOrder` | string | `asc` 或 `desc` |

## 6. 状态与动作约定

- 状态判断统一使用 `status`，页面展示使用 `statusName`。
- 前端不要用中文状态判断业务逻辑。
- 审批类动作统一使用：

```text
POST /api/v1/{resource}/{id}/submit
POST /api/v1/{resource}/{id}/approve
POST /api/v1/{resource}/{id}/reject
POST /api/v1/{resource}/{id}/revoke
```

销售订单常用动作：

```text
POST /api/v1/sales-orders/{id}/confirm
POST /api/v1/sales-orders/{id}/cancel
POST /api/v1/sales-orders/{id}/close
POST /api/v1/sales-orders/{id}/create-delivery
POST /api/v1/sales-orders/{id}/create-invoice
POST /api/v1/sales-orders/{id}/create-receipt
```

## 7. Mock 切换约定

- mock 字段必须覆盖页面契约中的关键字段。
- 后端首版接口可以先返回 mock 同结构数据。
- mock 到真实接口切换只改 API adapter，不改页面组件字段。
- 销售中心建议拆分 mock：

```text
mock/sales/
  customers.json
  sales-plans.json
  sales-quotes.json
  sales-contracts.json
  sales-orders.json
```

## 8. 联调验收清单

- 客户、计划、报价、合同、订单列表可分页、搜索、筛选。
- 详情接口字段覆盖页面详情、明细行、操作记录。
- 新增、编辑、提交、审批、确认、关闭等动作有明确返回状态。
- 金额、信用额度、应收、开票、回款字段使用 number。
- 销售退货、销售换货、销售报表如暂不开发，应返回明确的预留范围和后续版本计划。
