# 生产中心前后端接口协作规范

适用模块：生产中心  
覆盖页面：生产需求、生产计划、生产订单、生产工单、生产排班、委外加工  
接口前缀：`/api/v1`

> 说明：本文件定义生产中心前后端协作的通用约束；具体字段、枚举、接口路径见《生产中心-页面接口契约.md》。

## 1. 协作目标

- 前端 mock、生产页面、后端接口围绕同一套需求、计划、订单、工单、排班、委外字段协作。
- 生产状态、BOM/工艺路线锁版、工序进度、报工、质检、入库记录需要能前后追溯。
- 联调验收以生产中心页面契约为准，跨模块数据与销售、仓储、质检中心保持编号和状态一致。

## 2. 基础约定

| 项 | 约定 |
| --- | --- |
| API 前缀 | `/api/v1` |
| 数据格式 | `application/json` |
| 字符编码 | `UTF-8` |
| 日期 | `YYYY-MM-DD` |
| 日期时间 | ISO 8601，例如 `2026-06-07T10:12:00+08:00` |
| 字段命名 | `camelCase` |
| 主键 | `id` |
| 业务编号 | `code` |
| 数量 | number，单位单独返回 `unitName` |
| 工时 | number，单位建议使用小时 |

## 3. 生产中心资源命名

| 页面 | 推荐资源路径 | 说明 |
| --- | --- | --- |
| 生产需求 | `/production-demands` | 需求来源、产品、数量、交付日期 |
| 生产计划 | `/production-plans` | 计划周期、齐套、排产、审批 |
| 生产订单 | `/production-orders` | 生产数量、BOM/工艺锁版、车间产线 |
| 生产工单 | `/production-work-orders` | 工序、派工、报工、质检节点 |
| 生产排班 | `/production-schedules` | 班组、班次、日历、冲突校验 |
| 委外加工 | `/outsource-orders` | 委外发料、收货、质检、入库 |

## 4. 统一响应结构

成功响应：

```json
{
  "success": true,
  "data": {},
  "message": ""
}
```

列表响应：

```json
{
  "success": true,
  "data": {
    "items": [],
    "page": {
      "pageNo": 1,
      "pageSize": 20,
      "total": 156,
      "pages": 8
    }
  },
  "message": ""
}
```

错误响应：

```json
{
  "success": false,
  "code": "PRODUCTION_ORDER_NOT_FOUND",
  "message": "生产订单不存在",
  "details": {}
}
```

## 5. 列表查询约定

通用 Query 参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `pageNo` | number | 页码，从 1 开始 |
| `pageSize` | number | 每页条数 |
| `keyword` | string | 全局搜索 |
| `status` | string | 状态过滤，多个值用逗号分隔 |
| `productId` | string | 产品过滤 |
| `workshopId` | string | 车间过滤 |
| `lineId` | string | 产线过滤 |
| `teamId` | string | 班组过滤 |
| `sourceType` | string | 来源类型过滤 |
| `createdFrom` | string | 创建日期起 |
| `createdTo` | string | 创建日期止 |

## 6. 状态与生产动作约定

- 状态判断统一使用 `status`，页面展示使用 `statusName`。
- 生产数量、合格数量、不良数量、已完工数量必须使用 number。
- 报工动作应返回工单最新进度，并能生成待质检记录或成品入库记录。

常用动作：

```text
POST /api/v1/production-demands/{id}/confirm
POST /api/v1/production-demands/{id}/create-plan
POST /api/v1/production-plans/{id}/submit
POST /api/v1/production-plans/{id}/approve
POST /api/v1/production-plans/{id}/create-order
POST /api/v1/production-orders/{id}/release
POST /api/v1/production-orders/{id}/close
POST /api/v1/production-work-orders/{id}/dispatch
POST /api/v1/production-work-orders/{id}/start
POST /api/v1/production-work-orders/{id}/report
POST /api/v1/outsource-orders/{id}/issue-material
POST /api/v1/outsource-orders/{id}/receive
```

## 7. Mock 切换约定

- mock 必须包含来源单据、产品、数量、计划日期、状态、责任人。
- 生产订单和工单 mock 要包含明细行、工序、BOM/工艺版本信息。
- 生产中心建议拆分 mock：

```text
mock/manufacturing/
  demands.json
  plans.json
  orders.json
  work-orders.json
  schedules.json
  outsource-orders.json
```

## 8. 联调验收清单

- 生产需求可转生产计划或生产订单。
- 生产计划可审批、齐套检查、生成生产订单。
- 生产订单能展示 BOM/工艺锁版、工序进度、领退料、入库、质检记录。
- 生产工单支持派工、开工、报工、暂停、完工。
- 排班支持班组、班次、日历、冲突校验。
- 委外加工支持发料、收货、质检、入库闭环。
