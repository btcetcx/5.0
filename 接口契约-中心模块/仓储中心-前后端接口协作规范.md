# 仓储中心前后端接口协作规范

适用模块：仓储中心  
覆盖页面：库存管理、出库管理、入库管理、调拨管理、盘点管理、出库质检、来料质检、仓库库位  
接口前缀：`/api/v1`

> 说明：本文件定义仓储中心前后端协作的通用约束；具体字段、枚举、接口路径见《仓储中心-页面接口契约.md》。

## 1. 协作目标

- 前端页面、mock 数据、扫码/库位/批次交互与后端接口字段保持一致。
- 库存数量、冻结数量、锁定数量、在途数量、质量状态、成本层等关键字段以后端为准。
- 联调时重点核对出入库、调拨、盘点对库存流水和库存余额的影响。

## 2. 基础约定

| 项 | 约定 |
| --- | --- |
| API 前缀 | `/api/v1` |
| 数据格式 | `application/json` |
| 字符编码 | `UTF-8` |
| 日期 | `YYYY-MM-DD` |
| 日期时间 | ISO 8601，例如 `2026-06-07T10:12:00+08:00` |
| 字段命名 | `camelCase` |
| 主键 | `id` |
| 业务编号 | `code` |
| 数量 | number，单位单独返回 `unitName` |
| 金额 | number，币种单独返回 `currency` |

## 3. 仓储中心资源命名

| 页面 | 推荐资源路径 | 说明 |
| --- | --- | --- |
| 库存管理 | `/warehouse-stocks` | 物品库存、批次、库位、冻结、质量状态 |
| 入库管理 | `/warehouse-inbounds` | 入库单、待入库、入库明细、上架 |
| 出库管理 | `/warehouse-outbounds` | 出库单、待出库、拣货、复核、发货 |
| 调拨管理 | `/warehouse-transfers` | 仓间调拨、库位调拨、差异处理 |
| 盘点管理 | `/warehouse-inventories` | 盘点计划、盘点单、差异调整 |
| 仓库库位 | `/warehouse-locations` | 仓库、区域、库位、容量 |
| 出库质检 | `/warehouse-quality-inspections` | 预留，出库质检任务 |
| 来料质检 | `/warehouse-quality-inspections` | 预留，来料质检任务 |

## 4. 统一响应结构

成功响应：

```json
{
  "success": true,
  "data": {},
  "message": ""
}
```

列表响应：

```json
{
  "success": true,
  "data": {
    "items": [],
    "page": {
      "pageNo": 1,
      "pageSize": 20,
      "total": 156,
      "pages": 8
    }
  },
  "message": ""
}
```

错误响应：

```json
{
  "success": false,
  "code": "WAREHOUSE_STOCK_NOT_ENOUGH",
  "message": "可用库存不足",
  "details": {}
}
```

## 5. 列表查询约定

通用 Query 参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `pageNo` | number | 页码，从 1 开始 |
| `pageSize` | number | 每页条数 |
| `keyword` | string | 全局搜索 |
| `status` | string | 状态过滤，多个值用逗号分隔 |
| `warehouseId` | string | 仓库过滤 |
| `locationId` | string | 库位过滤 |
| `materialId` | string | 物品过滤 |
| `batchNo` | string | 批次过滤 |
| `qualityStatus` | string | 质量状态过滤 |
| `createdFrom` | string | 创建日期起 |
| `createdTo` | string | 创建日期止 |

## 6. 状态与库存动作约定

- 状态判断统一使用 `status`，页面展示使用 `statusName`。
- 数量字段不得返回中文单位拼接字符串。
- 影响库存的动作必须返回最新单据状态，并能追溯库存流水。

常用动作：

```text
POST /api/v1/warehouse-inbounds/{id}/confirm
POST /api/v1/warehouse-inbounds/{id}/putaway
POST /api/v1/warehouse-outbounds/{id}/confirm
POST /api/v1/warehouse-outbounds/{id}/pick
POST /api/v1/warehouse-outbounds/{id}/ship
POST /api/v1/warehouse-transfers/{id}/submit
POST /api/v1/warehouse-transfers/{id}/out-confirm
POST /api/v1/warehouse-transfers/{id}/in-confirm
POST /api/v1/warehouse-inventories/{id}/confirm
POST /api/v1/warehouse-inventories/{id}/adjust
```

## 7. Mock 切换约定

- mock 必须包含库存数量、可用数量、冻结数量、库位、批次、质量状态。
- 出入库、调拨、盘点 mock 要保留明细行，不能只有单据头。
- 仓储中心建议拆分 mock：

```text
mock/warehouse/
  stocks.json
  inbounds.json
  outbounds.json
  transfers.json
  inventories.json
  locations.json
```

## 8. 联调验收清单

- 库存列表可按仓库、库位、批次、质量状态筛选。
- 入库确认后能看到库存流水和库存余额变化。
- 出库确认前后可用量、锁定量、已出库量一致。
- 调拨支持调出、在途、调入、差异处理状态。
- 盘点支持锁库、实盘、复盘、差异调整。
- 出库质检、来料质检如暂不开发，应明确为预留入口。
